# 設計判断の経緯

「なぜそうしたか」を記録する。CSS変数名の謎、セキュリティ対応の経緯など、
あとから「これは仕様？バグ？」と迷いやすい判断の理由をまとめる。

## CSS / カラーテーマ（2026-05-19）

カラーテーマを緑→ピンクに変更したが、CSS変数名は --green / --green-light のまま残している。

- `--green` の実値は `#D4537E`（濃ピンク・テーマカラー）
- `--green-light` の実値は `#F4C0D1`（中ピンク）
- `--pink` の実値は `#D4537E`（`--green` と同値、体重グラフから残存）
- `--cat-life` も `#D4537E`（テーマカラーと同値）
- 年タブ選択中の border-bottom も `--green`

変数名と実値が一致していないが、これは意図的な設計判断:
- 参照箇所が42箇所あり、リネームのリスクを避けるため変数名はそのまま
- 値だけ書き換えて全体を一斉にピンク化
- 「緑なのにピンク」に見えるが、これは仕様であってバグではない

将来、変数名を `--pink-main` などにリネームする余地はある
（その場合は42箇所の参照を全て書き換える必要あり）。

## gas_url を同期・復元から保護する理由（2026-05-28）

GAS の Web App URL（`gas_url`）は、同期と復元の対象から意図的に外している。
これは 2026-05-28 のセキュリティ対応（GAS URL を index.html へのハードコードから
localStorage の `gas_url` キー管理に変更した対応）の一環で確立した設計判断。

- **`gas_url` は同期対象外**
  - `gas_url` は `pushAllToCloud` / `syncFromCloud` の送受信対象から除外している
    （送らない・上書きされない）。
  - `gas_url` は端末ごとに個別設定する接続先情報であり、健康データそのものではない。
    同期に含めると別端末の設定で上書きされうるため、除外していると考えられる。
- **`restoreFromBackup` でも `gas_url` を保護**
  - バックアップから復元するときも `gas_url` を上書きしないよう保護している。
  - 現行URLは各端末の localStorage に個別設定する必要があるため、
    復元で書き換わらないようにする意図と考えられる。

対応の全体記録（旧→新の経緯・設定UI・旧URL失効など）は history.md の
「2026-05-28 セキュリティ対応・カレンダー改善」→「GAS URLのlocalStorage化」を参照。
