（過去に確立した実装パターン：検索UI・配列更新・排他制御など。※後日移行）

### 編集時の配列更新パターン
- `arr.map` で位置を保持する（findIndex + splice は順序が崩れる場合あり）

### 排他制御の完全版
- 「追加を開く」時は edit と expanded の両方を null にする
- どちらか片方だけだと、別エントリの編集中に追加フォームが開いて競合する

### 検索機能の設計（健康ノート・あおの記録で確立）
- 検索ボックスのDOMは「初回のみ作成」「以降は触らない」
- renderXxx 内で毎回 innerHTML で再構築するとIMEが壊れる
- エントリリストだけ毎回書き換え、検索ボックス・件数バッジ・✕ボタンは個別IDで更新
- IME対応（漢字変換）はDOM再構築問題が解決すれば自動的に動く
- ハイライト用クラス: 健康ノート `.note-hit` / あおの記録 `.ao-hit`（薄ピンク）
- 状態変数: `_noteSearchQuery` / `_aoRecordSearchQuery`、デバウンス用 `_xxxSearchDebounce`
- フィルタ順序: ソート → 年 → カテゴリ → 検索 → 描画
- 検索フィルタは text のみで十分（text に元気度・特記事項などが結合保存されているため）
- デバッグログは prefix（例 `[NoteSearch]` `[AoSearch]`）で統一・原因確定後に削除
- 入力UIを含むカードは、最初から「入力要素のDOMは再構築しない」設計を採用する

### フィルタ追加時の自動リセット思想
- 検索開始時: 年タブを「全期間」・カテゴリチップを全ONにリセット
- 検索クリア時: 文字だけ消す。年タブ・カテゴリチップのフィルタは直前の状態維持

### タブ並び順の同期注意点
- HTMLタブ順を変更したら `TAB_NAMES` 配列（line 3832 付近）も同じ順に同期必須
- `switchTab` が DOMインデックスと `TAB_NAMES` のインデックスで照合しているため

### 月切替ナビ
- `.notes-nav-label.clickable` のピル形スタイル
- 「全期間」リセットは中央ラベルをピル形ボタン化（案A）
- 慣れるまで「押せる」と気付かれにくい可能性あり、必要に応じて案Bへ変更可能

### text 結合・parse の制約
- `saveTodayAll` 行1565-1571 の text 結合ロジック
- text → フィールドの parse は元の構造に依存するため、フィールドを増やす時は要注意
- 健康ノート編集の「方針W」の設計

### 削除時の整合パターン
- diary / day_memo の整合を保つために両方を更新する

### サプリの完全削除手段（2026-05-21 追加）
- 編集フォーム内の「このサプリを削除」リンクから削除可能
- confirm ダイアログでサプリ名を表示して誤削除防止
- 同期挙動: 削除はローカルのみ反映、別端末には伝播しない（tombstone 未実装）
- 別端末でも同じサプリを削除したい場合は、その端末でも同じ操作を行う
- 用途: テスト用・誤追加サプリの片付けが中心。日常的な「服用をやめた」管理は `status:'stopped'` で行う
